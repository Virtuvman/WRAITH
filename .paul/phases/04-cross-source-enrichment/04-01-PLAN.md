<objective>
## Goal
Create two pure data modules — modules/shodan.py (Shodan free-tier IP/geo lookup) and modules/osm.py (Overpass API POI query) — that enrich WiGLE selector observations with nearby exposed-device intelligence and geospatial context.

## Purpose
Phase 3 established the selector tracking foundation. Phase 4 adds signal depth: Shodan surfaces exposed IP infrastructure near WiGLE observations (cameras, routers, industrial devices), while OSM/Overpass adds behavioral context (cafes, government buildings, military installations) near selector coordinates. Together they give the operator the cross-source picture needed to assess D-ISR and physical-ISR vulnerability.

## Output
- modules/shodan.py — shodan_host_lookup, shodan_search_near, enrich_with_shodan, mock fallback
- modules/osm.py — overpass_pois, session cache, POI DataFrame schema
</objective>

  <name>Task 1: Create modules/shodan.py</name>
  <files>modules/shodan.py</files>
  <action>
    Create a pure data module (no streamlit imports) with these functions:

    **SHODAN_RESULT_COLUMNS** constant:
    `["ip", "ports", "org", "product", "country", "lat", "lon"]`

    **shodan_host_lookup(ip: str, api_key: str, mock: bool = False) -> dict:**
    - If mock=True: return realistic fake dict with ip, ports=[80,443], org="Mock ISP", hostnames=[], vulns=[], error=None
    - GET https://api.shodan.io/shodan/host/{ip}?key={api_key}
    - On 200: parse and return {ip, ports (list of int), org (str), hostnames (list), vulns (list of CVE strings from data.vulns.keys()), error: None}
    - On 401: return {error: "Invalid API key"}
    - On 404: return {error: "Host not found"}
    - On 429: return {error: "Rate limit — wait 1s and retry"}
    - On any exception: return {error: str(e)}
    - Never raise

    **shodan_search_near(lat: float, lon: float, radius_km: float, api_key: str, limit: int = 10, mock: bool = False) -> pd.DataFrame:**
    - If mock=True: return generate_mock_shodan(lat, lon, n=limit)
    - If not api_key: return pd.DataFrame(columns=SHODAN_RESULT_COLUMNS)
    - GET https://api.shodan.io/shodan/search?key={api_key}&query=geo:{lat},{lon},{radius_km}&page=1
    - On 200: normalize matches list into DataFrame with SHODAN_RESULT_COLUMNS
      - ip from "ip_str", ports from "ports", org from "org" (or "isp"), product from "product" (or ""), country from "location.country_name", lat/lon from "location.latitude/longitude"
    - On any non-200 or exception: return empty DataFrame, log warning
    - Never raise

    **generate_mock_shodan(lat: float, lon: float, n: int = 5) -> pd.DataFrame:**
    - Generate n rows with realistic device types (Hikvision camera, MikroTik router, Siemens PLC, nginx server, Apache httpd)
    - Scatter lat/lon ±0.005 degrees from input
    - Return DataFrame with SHODAN_RESULT_COLUMNS

    **enrich_with_shodan(df: pd.DataFrame, api_key: str, mock: bool = False) -> pd.DataFrame:**
    - Accepts a wigle_networks-schema DataFrame
    - Adds columns: shodan_nearby (bool, whether Shodan devices found within 0.01 deg), shodan_count (int)
    - For each unique (trilat, trilong) coordinate: call shodan_search_near with radius_km=1
    - If result is empty: shodan_nearby=False, shodan_count=0
    - If df is None or empty: return empty DataFrame
    - Never raise

    Avoid: importing streamlit, requests sessions that persist state between module loads.
    Use: standard requests library (already in WRAITH stack).
  </action>

<task type="auto">
  <name>Task 2: Create modules/osm.py</name>
  <files>modules/osm.py</files>
  <action>
    Create a pure data module (no streamlit imports) with these functions:

    **OSM_RESULT_COLUMNS** constant:
    `["name", "amenity", "lat", "lon", "osm_id", "tags"]`

    **DEFAULT_POI_TAGS** constant — list of Overpass tag filters to query by default:
    `["amenity", "military", "office", "government", "shop", "landuse=military"]`

    **_osm_cache: dict[tuple, pd.DataFrame] = {}** — module-level session cache

    **overpass_pois(lat: float, lon: float, radius_m: float = 500, tags: list[str] | None = None, timeout: int = 10) -> pd.DataFrame:**
    - Cache key: (round(lat, 3), round(lon, 3), int(radius_m))
    - On cache hit: return cached result
    - Build Overpass QL query — union of node/way/relation for each tag within radius_m of lat/lon:
      ```
      [out:json][timeout:{timeout}];
      (
        node["amenity"](around:{radius_m},{lat},{lon});
        node["military"](around:{radius_m},{lat},{lon});
        node["office"](around:{radius_m},{lat},{lon});
        node["shop"](around:{radius_m},{lat},{lon});
      );
      out center;
      ```
      Use tags param if provided, else DEFAULT_POI_TAGS (only use simple tag names, not "landuse=military" in node filter — convert "key=value" to node["key"="value"] form)
    - POST to https://overpass-api.de/api/interpreter with data={"data": query_string}
    - Set User-Agent header: "WRAITH-OSINT/1.0 (research)"
    - On 200: parse response JSON elements list
      - For each element: extract name from tags.get("name",""), amenity from tags.get("amenity",""), lat/lon (use "lat"/"lon" for nodes, "center.lat"/"center.lon" for ways/relations), osm_id from "id", tags as str(element.get("tags",{}))
    - On timeout (requests.Timeout) or any exception: log warning, return empty DataFrame
    - Store result in cache before returning
    - Never raise

    **clear_osm_cache() -> None:** clears _osm_cache dict

    Avoid: importing streamlit. Do not use session_state.
    Use: requests library, standard timeout parameter to avoid hanging on slow Overpass responses.
    Note: Overpass can be slow — timeout=10 is a reasonable default for single-coordinate queries.
  </action>
  <verify>python -m py_compile modules/osm.py — exit 0, no errors</verify>
  <done>AC-4, AC-5 satisfied: module compiles, overpass_pois returns correct schema and caches results</done>
