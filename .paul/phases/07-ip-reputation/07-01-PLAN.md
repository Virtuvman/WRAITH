<objective>
## Goal
Create modules/report.py — a pure data module that takes the current Selector view state (filtered observations, co-location, velocity, entity data) and renders a self-contained HTML intelligence report, plus a CSV export of raw observations.

## Purpose
Every investigation in WRAITH-WiGLE v0.1 exists only in the browser session — closing the app loses the picture. The intel report captures the full ISR chain into a shareable, self-contained HTML file that can be sent to a colleague, attached to a report, or archived. No API keys. No new dependencies beyond Jinja2 (already installed via Streamlit).

## Output
- modules/report.py — generate_html_report, generate_csv_export, REPORT_SECTIONS

  <action>
    Create a pure data module (no streamlit imports) using Jinja2 for HTML rendering.

    **Imports:**
    ```python
    import datetime
    import pandas as pd
    from jinja2 import Template
    ```

    **REPORT_SECTIONS constant:**
    ```python
    REPORT_SECTIONS = [
        "observations", "colocation", "velocity",
        "shodan", "osm", "telegram",
    ]
    ```

    **_df_to_html(df, empty_msg="No data available.") -> str:**
    - If df is None or df.empty: return f'<p class="empty">{empty_msg}</p>'
    - Return df.to_html(index=False, border=0, classes="data-table")

    **_anomaly_summary(vel_df) -> str:**
    - If vel_df is None or vel_df.empty or "is_anomaly" not in vel_df.columns: return "No velocity data."
    - Count anomalies: n = int(vel_df["is_anomaly"].sum())
    - Return f"{n} anomalous movement(s) detected." if n > 0 else "No anomalous movements detected."

    **HTML_TEMPLATE constant** — inline Jinja2 template string:
    ```
    A clean dark-theme HTML report with:
    - Header: "WRAITH-WiGLE // SELECTOR INTELLIGENCE REPORT"
    - Meta: selector field, selector value, generated timestamp, observation count
    - Sections (each with heading + table or empty message):
        Observations (lasttime, trilat, trilong, ssid, netid, encryption, near_camera)
        Co-located Networks (netid, ssid, type, encryption, coloc_count)
        Movement Analysis (anomaly summary text + speed table)
        Shodan Devices (ip, ports, org, product, country)
        OSM Points of Interest (name, amenity, lat, lon)
        Telegram Mentions (channel, date, text, url)
    - Footer: "WRAITH-WiGLE v0.2 — For authorized OSINT use only."
    - Inline CSS: dark background (#0f172a), white text, monospace font,
      table borders (#334155), section headers in accent color (#38bdf8)
      Anomaly rows: class "anomaly" styled red
    - No external CSS/JS links — fully self-contained
    ```

    Use Python triple-quoted string for the template — no external .html file needed.

    **generate_html_report(selector_field, selector_value, filtered_df, coloc_df=None, vel_df=None, shodan_df=None, osm_df=None, tg_df=None) -> str:**
    - Build context dict with:
        selector_field, selector_value
        generated_at = datetime.datetime.utcnow().strftime("%Y-%m-%d %H:%M UTC")
        obs_count = len(filtered_df) if filtered_df is not None else 0
        obs_cols = ["lasttime","trilat","trilong","ssid","netid","encryption","near_camera"]
        obs_html = _df_to_html(filtered_df[obs_cols] if filtered_df is not None and not filtered_df.empty and all(c in filtered_df.columns for c in obs_cols) else filtered_df)
        coloc_cols = ["netid","ssid","type","encryption","coloc_count"]
        coloc_html = _df_to_html(...)
        vel_cols = ["lasttime","speed_kmh","distance_km","is_anomaly"]
        vel_html = _df_to_html(...)
        anomaly_summary = _anomaly_summary(vel_df)
        shodan_html, osm_html, tg_html via _df_to_html
    - Render HTML_TEMPLATE with context
    - Return rendered string
    - On any exception: log warning, return minimal error HTML string — never raise

    **generate_csv_export(filtered_df) -> str:**
    - If filtered_df is None or filtered_df.empty: return column headers only as CSV string
    - Return filtered_df.to_csv(index=False)
    - Never raise

    Avoid: importing streamlit. No file I/O in this module — callers handle saving.
