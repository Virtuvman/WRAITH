<objective>
## Goal
Add source filter and file upload to the RAVEN view: a multiselect that hides/shows pins by source, and a file uploader that accepts CSV and Shodan/FOFA JSON files and merges them into the live map without reloading feed data.

## Purpose
The map is live but unfiltered — with 65+ pins across 4–5 sources, operators need to isolate sources quickly. File upload completes the Phase 9 promise: load_raven_csv was built to accept file-like objects for exactly this; JSON upload requires a small load_raven_file extension.

## Output
- modules/raven_ingest.py — load_raven_file accepts filepath string OR file-like object (BytesIO/StringIO)
- app.py — source filter multiselect + file uploader wired into render_raven_view(); two new session state keys
</objective>
